# PDF Analyzer MVP 需求文档

**项目名称**：PDF Analyzer  
**版本**：v1.0 MVP  
**创建日期**：2026-03-17  
**状态**：待开发

---

## 1. 项目概述

### 1.1 产品定位
一个基于 AI 的 PDF 文档分析工具，用户上传 PDF 文件后，AI 自动提取内容并进行智能分析（如摘要、关键信息提取、问答等）。

### 1.2 目标用户
- 需要快速理解长文档的用户
- 学生、研究人员、企业员工
- 任何需要处理大量 PDF 文档的人

### 1.3 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端框架 | **Streamlit** | 快速构建数据应用 UI |
| AI 模型 | **MiniMax M2.5** | 文档分析与理解 |
| PDF 解析 | **PyMuPDF (fitz)** | 提取 PDF 文本内容 |
| 部署平台 | **Streamlit Cloud** | 免费托管 Streamlit 应用 |
| 环境管理 | Python 3.11+ / pip | 依赖管理 |

---

## 2. 功能需求

### 2.1 核心功能 (MVP)

#### F1: PDF 上传
- 支持上传单个 PDF 文件
- 文件大小限制：最大 10MB
- 支持拖拽上传和点击选择

#### F2: 文本提取
- 自动提取 PDF 中的文字内容
- 支持多页文档
- 保留基本段落结构

#### F3: AI 分析功能
- **智能摘要**：生成文档概要（200-500字）
- **问答交互**：用户可以针对文档内容提问
- **关键信息提取**：自动提取关键字段（标题、日期、金额等）

#### F4: 结果展示
- 清晰展示分析结果
- 支持复制结果内容

### 2.2 扩展功能 (后续版本)

- 多文件批量处理
- 支持扫描件/图片 PDF（OCR）
- 自定义分析 prompt
- 分析历史记录
- 导出分析报告

## 5. API 配置

### 5.1 MiniMax API

| 参数 | 值 |
|------|-----|
| API Key | (用户自行配置) |
| Base URL | `https://api.minimax.chat/v1` |
| 模型 | `MiniMax-M2.5` |
| 调用方式 | Chat Completion API |

### 5.2 环境变量

```bash
# .streamlit/secrets.toml
MINIMAX_API_KEY = "your-api-key-here"
MINIMAX_BASE_URL = "https://api.minimax.chat/v1"
```

## 7. 部署方案

### 7.1 Streamlit Cloud 部署

1. **准备代码**
   - 推送到 GitHub 仓库
   - 包含 `requirements.txt` 和 `app.py`

2. **配置 secrets**
   - 在 Streamlit Cloud 后台配置 `MINIMAX_API_KEY`

3. **部署流程**
   - New App → 关联 GitHub 仓库 → Deploy

### 7.2 目录结构

```
pdf-analyzer/
├── app.py                 # 主应用
├── requirements.txt       # 依赖

├── utils/
```
